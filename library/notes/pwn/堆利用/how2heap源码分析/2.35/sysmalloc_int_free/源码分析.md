# 一、`sysmalloc` 是干什么的？

一句话定义：

> **当当前 arena 的 top chunk 不够用时，向系统申请新内存（sbrk / mmap）**

触发条件在 `_int_malloc` 中：

`if (size > chunksize (av->top))     p = sysmalloc (nb, av);`

---

# 二、函数参数与变量

`static void * sysmalloc (INTERNAL_SIZE_T nb, mstate av)`

- `nb`：**已经对齐过的请求 chunk 大小（含 header）**
    
- `av`：当前 arena（`main_arena` 或 thread arena）
    

---

## 关键局部变量

`mchunkptr old_top;      // 旧的 top chunk INTERNAL_SIZE_T old_size; char *old_end;          // old_top 的末尾地址`

> 记录 **扩展前的堆状态**

---

`long size; char *brk; char *snd_brk;`

- `size`：sbrk 要申请的大小
    
- `brk`：第一次 sbrk 返回值
    
- `snd_brk`：第二次 sbrk（用于对齐/修正）
    

---

`INTERNAL_SIZE_T front_misalign; INTERNAL_SIZE_T end_misalign; char *aligned_brk;`

👉 **处理 MALLOC_ALIGNMENT 对齐**

---

`mchunkptr p; mchunkptr remainder;`

- `p`：最终返回给用户的 chunk
    
- `remainder`：新的 top chunk
    

---

# 三、第一阶段：是否直接 mmap？

`if (av == NULL     || ((unsigned long)(nb) >= mp_.mmap_threshold         && (mp_.n_mmaps < mp_.n_mmaps_max)))`

### 条件解释

|条件|含义|
|---|---|
|`av == NULL`|没有 arena（极端失败路径）|
|`nb >= mmap_threshold`|大 chunk|
|`n_mmaps < n_mmaps_max`|mmap 数量未超限|

👉 **大块优先 mmap**

---

### Huge page mmap（THP）

`mm = sysmalloc_mmap (nb, mp_.hp_pagesize, mp_.hp_flags, av);`

- 使用 huge page
    
- 成功直接 `return`
    

---

### 普通 mmap

`mm = sysmalloc_mmap (nb, pagesize, 0, av);`

> **成功就结束，完全绕过 heap**

📌 **利用点提示**

- mmap chunk **不进 bins**
    
- 不参与 unlink
    
- UAF/overlap 模型不同
    

---

# 四、arena 不存在 → 直接失败

`if (av == NULL)     return 0;`

---

# 五、记录旧 top 状态

`old_top = av->top; old_size = chunksize(old_top); old_end = chunk_at_offset(old_top, old_size);`

---

### 关键断言（堆完整性）

`assert ((old_top == initial_top (av) && old_size == 0) ||         (old_size >= MINSIZE &&          prev_inuse(old_top) &&          old_end page-aligned));`

👉 **防止堆被破坏**

---

# 六、非 main_arena（thread arena）

`if (av != &main_arena)`

## 1️⃣ 尝试扩展当前 heap

`grow_heap(old_heap, MINSIZE + nb - old_size)`

- 本质：`mremap` 或 `sbrk`
    
- 成功：
    

`set_head(old_top, new_size | PREV_INUSE);`

---

## 2️⃣ 新建 heap（heap_info）

`heap = new_heap(nb + MINSIZE + sizeof(heap_info))`

### 发生了什么？

- 分配新 heap region
    
- old heap 被 fencepost 隔离
    
- old top 被 free
    

`_int_free(av, old_top, 1);`

📌 **这一步是很多 heap exploit 的“隐形 free”来源**

---

## 3️⃣ new_heap 失败 → fallback mmap

`sysmalloc_mmap(nb)`

---

# 七、main_arena（重点）

`else /* av == main_arena */`

---

## 1️⃣ 计算 sbrk 大小

`size = nb + mp_.top_pad + MINSIZE;`

- `top_pad`：预留空间，减少系统调用
    
- `MINSIZE`：保证新 top chunk 合法
    

---

### contiguous 优化

`if (contiguous(av))     size -= old_size;`

👉 如果是连续 heap，旧 top 可以合并

---

### 页对齐

`size = ALIGN_UP(size, pagesize);`

---

## 2️⃣ 第一次 sbrk

`brk = MORECORE(size);`

- 成功：heap 向上增长
    
- 失败：进入 mmap fallback
    

---

## 3️⃣ sbrk 失败 → mmap fallback

`sysmalloc_mmap_fallback(...)`

👉 **把 mmap 当成“伪 sbrk”**

---

## 4️⃣ sbrk 成功后的三种情况

---

### ✅ 情况一：完美连续

`if (brk == old_end)     set_head(old_top, old_size + size | PREV_INUSE);`

👉 **最理想**

---

### ❌ 情况二：heap 被外部 sbrk 干扰

`malloc_printerr("break adjusted...");`

👉 **直接 abort**

---

### 🔧 情况三：需要对齐 / 修正（最复杂）

#### 处理：

- MALLOC_ALIGNMENT
    
- foreign sbrk
    
- page boundary
    
- second sbrk
    

`snd_brk = MORECORE(correction);`

---

### 设置新 top

`av->top = aligned_brk; set_head(av->top, new_size | PREV_INUSE);`

---

### fencepost 插入（关键）

`set_head(old_top + old_size, CHUNK_HDR_SZ | PREV_INUSE);`

👉 **防止非法 consolidate**

📌 **利用点：**

- fencepost = fake inuse chunk
    
- 防止向不属于 allocator 的区域合并
    

---

# 八、最终真正分配 chunk

`p = av->top; size = chunksize(p);`

---

## 切分 top chunk

`remainder_size = size - nb; remainder = p + nb; av->top = remainder;`

---

### 设置 chunk 头

`set_head(p, nb | PREV_INUSE); set_head(remainder, remainder_size | PREV_INUSE);`

---

### 返回给用户

`return chunk2mem(p);`

---

# 九、失败兜底

`__set_errno(ENOMEM); return 0;`
