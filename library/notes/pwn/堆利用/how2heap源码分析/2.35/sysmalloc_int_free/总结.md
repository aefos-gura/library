# sysmalloc 触发 _int_free 的利用笔记（整理版）

> 目标：**在不直接调用 free() 的情况下**，通过 `malloc()` → `sysmalloc()` 的路径，让 **top chunk（wilderness）被 `_int_free`**，从而把**指定大小**的 chunk 放入 bins。
> 
> 典型应用：House of Orange / House of Tangerine

---

## 一、利用核心结论

> **当当前 arena 的 top chunk 无法与新申请到的内存合并时，`sysmalloc()` 会对旧 top chunk 调用 `_int_free()`。**

因此：

- 只要**控制 top chunk 的 size**，
    
- 并触发一次 **“top 不够用”的 malloc**，
    
- 就能把“伪造大小”的 top chunk **free 进任意 bin**。
    

---

## 二、触发条件（必要条件）

### 1️⃣ 能力假设

满足以下之一即可：

- 能 **修改 top chunk 的 size**（BOF / OOB / UAF / 任意写）
    
- 或者：能通过分配模式 **间接影响 top size**（较难）
    

> ⚠️ 本利用**不要求直接调用 free()**，但**sysmalloc 内部会隐式调用 `_int_free()`**。

---

### 2️⃣ 关键约束

- **top chunk size 必须 page-aligned**（通常 0x1000 对齐）
    
- 伪造后的 top size ≥ `MINSIZE + fencepost`
    
- arena = **main_arena**（thread arena 走 `new_heap`，模型不同）
    

---

## 三、攻击流程（标准 3 步）

### Step 1️⃣：控制 / 伪造 top chunk size

将 top chunk size 改成：

```
fake_top_size = target_free_chunk_size + FENCEPOST
```

在 glibc 2.27+：

```
FENCEPOST = 2 * CHUNK_HDR_SZ = 0x20 (x64)
```

并保证：

```
fake_top_size % PAGESIZE == 0
```

---

### Step 2️⃣：触发 sysmalloc（top 不够）

调用：

```c
malloc(fake_top_size - 0x10);
```

为什么？

- `_int_malloc` 中判断：
    

```c
if (size > chunksize(av->top))
    p = sysmalloc(nb, av);
```

- 当前 top chunk **不够用** → 进入 `sysmalloc()`
    

---

### Step 3️⃣：sysmalloc 内部发生的事（关键）

在 `sysmalloc()` 中（main_arena 路径）：

1. sbrk / mmap 扩展失败或不可合并
    
2. 插入 **fencepost**
    
3. 对旧 top 执行：
    

```c
_int_free(av, old_top, 1);
```

➡️ **伪造大小的 top chunk 被 free 进 bins**

---

## 四、free 后的堆结构（抽象）

```
[ 已分配 chunk ]
[  被 free 的旧 top  ]  ← 进入 unsorted / small / large bin
[  fencepost (inuse) ]
[  新 top chunk      ]
```

随后：

```c
malloc(target_size - CHUNK_HDR_SZ);
```

即可 **重新分配到该 freed chunk**。

---

## 五、为什么这不是“真正的漏洞”？

- 这是 **glibc 设计行为**，不是 bug
    
- 依赖：
    
    - allocator 必须信任 top chunk 的 size
        
    - fencepost 只防 consolidate，不防 free
        

因此：

> **真正的漏洞是：你能控制 top chunk size。**

sysmalloc 只是“放大器”。

---

## 六、与 bins 的关系

- 被 free 的 top chunk：
    
    - 不走 fastbin
        
    - 直接进 **unsorted bin**
        
    - 后续根据 size 进入 small / large bin
        

👉 可以构造：

- overlap
    
- fake chunk
    
- largebin attack
    
- unsorted bin write
    

---

## 八、与 House of Orange 的关系

|House of Orange|本技巧|
|---|---|
|伪造 top size|✅|
|触发 sysmalloc|✅|
|隐式 _int_free|✅|
|unsorted bin 攻击|常见后续|

本笔记本质是 **House of Orange 的最小抽象模型**。
